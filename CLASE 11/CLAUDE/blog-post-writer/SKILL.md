# Blog Post Writer

Write blog posts that people actually finish reading. The job is two-fold: satisfy the search engine enough to rank, and satisfy the human enough to stay.

## Before writing, gather these inputs

If the user hasn't provided them, ask:

1. **Topic / working title** — what's the post about?
2. **Target keyword** — the main phrase they want to rank for (optional but useful)
3. **Audience** — who's reading this? (beginners, pros, a specific role)
4. **Angle or POV** — what's the unique take? If none, suggest 2-3 angles before writing.
5. **Desired length** — short (600-900), standard (1200-1800), long-form (2500+)
6. **CTA** — what should the reader do after reading?

Don't ask all at once. If you can infer from context (e.g., their business is SaaS, audience is likely founders), do that and confirm in one line.

## Structure

Use this skeleton unless the user requests otherwise:

1. **Hook** (1-3 sentences) — a specific pain, a surprising claim, a contrarian take, or a concrete scene. Never start with "In today's fast-paced world" or dictionary-definition openers.
2. **Promise** (1-2 sentences) — what the reader will walk away with.
3. **Body** — H2 sections, each answering one question or covering one subtopic. Use H3s sparingly. Include examples, numbers, and screenshots/visual callouts where useful.
4. **Conclusion** — one paragraph that reinforces the main takeaway, not a summary.
5. **CTA** — single, specific action.

## SEO hygiene (when a target keyword is provided)

- Include the keyword in the title, the first 100 words, at least one H2, and naturally throughout the body (no stuffing)
- Use related terms and synonyms — search engines reward topical depth, not keyword density
- Write the title in 50-60 characters
- Write a meta description in 140-160 characters (provide this separately at the end)
- Suggest 3-5 internal link anchor points where relevant links would fit

## Voice rules

- Short sentences beat long ones. Vary length, but lean short.
- Write to one person, not an audience. "You" not "readers."
- Cut filler: "it's important to note," "as you can see," "in conclusion."
- Use concrete nouns and active verbs. Replace "utilize" with "use," "leverage" with "use."
- If a sentence could be cut without losing meaning, cut it.

## What to avoid

- Generic intros that restate the title
- Listicles with bullets of 3-5 words each (padding, no substance)
- "Pros and cons" tables unless genuinely useful
- Ending with "So there you have it!" or similar
- AI tells: "delve," "navigate the landscape," "in the realm of," "unleash," "harness the power"

## Output format

Deliver:
1. Title (plus 2 alternatives)
2. Meta description
3. The full post in markdown
4. 3-5 suggested internal link anchors
