# Twitter/X Thread Writer

A thread is a series of tweets engineered to be read in order. The first tweet has to stop the scroll. Every subsequent tweet has to earn the next tap. Miss either and the whole thread dies in the algorithm.

## Step 1: Qualify the idea

Not every idea deserves a thread. Before writing, check:

- **Is this thread-worthy?** — A single tweet or a blog post might be better. Threads work for: tactical breakdowns, stories with beats, lists where each item has substance, arguments that build.
- **Does it have a clear payoff?** — The reader should feel they got something by the end.
- **Would a single tweet actually hit harder?** — If yes, do that instead.

## Step 2: Get the inputs

- **Core idea** — the single thing the thread is about
- **Audience** — who's reading and what do they already know?
- **Thread purpose** — idea sharing, driving to a link, building authority, engagement?
- **Voice** — any examples of the user's existing threads or tweets?
- **Length target** — 5-tweet, 10-tweet, or "as long as it needs to be"

## Thread structure

### Tweet 1 — the hook (the whole ballgame)
This is 90% of your thread's success. If it doesn't get the tap, the rest doesn't exist.

Length: Use all 280 chars strategically, or go very short — both work. Medium-length hooks get ignored.

Hook patterns that work:

- **Specific number + bold claim:** "I made $142,000 from one Twitter thread. Here's exactly how."
- **Contrarian:** "Everything you've been told about [X] is wrong. The actual truth:"
- **Before/after transformation:** "2 years ago I couldn't [X]. Today I [result]. The exact steps I took:"
- **Listicle tease:** "5 [things] that [unexpected outcome]. Steal these:"
- **Story promise:** "The weirdest sales call I ever had made me $50K. Here's what happened:"
- **Mistake confession:** "I spent 3 years doing [thing] wrong. If I started over, I'd do this:"

Formatting the hook:
- Line breaks matter — give the eye rest
- Often end with a colon to signal "more coming below"
- Can use arrows (↓) or emojis sparingly to pull the eye down

Avoid:
- "A thread 🧵" as the first line before the actual hook — burying the lede
- Vague questions ("What do you think about X?")
- Generic claims ("Mindset is everything")

### Tweets 2-N — the body

Each tweet is its own micro-unit. Rules:

- **One beat per tweet.** Don't cram two ideas into one tweet.
- **Open with momentum.** First line of each tweet is a mini-hook.
- **Short lines > long blocks.** White space is a feature.
- **Avoid "continued" or numbering if possible** — the format tells you it's sequential
- **Pattern variety** — don't make every tweet the same shape. Mix short punchy ones with meatier ones with lists.

Formats that work inside a thread:

- **Point + example + lesson** (three-beat tweet)
- **Claim + data** ("67% of X do Y. Why:")
- **Before / After** (side-by-side or sequential)
- **Negative space** (a single sentence on its own line for emphasis)

### Final tweet — the close + CTA

Two jobs:
1. Land the payoff — don't fizzle
2. One clear CTA

CTA options:
- "RT the first tweet if this was useful" (asks for reach)
- "Follow for more like this" (asks for follow)
- Link to your newsletter / product / further resource
- "Comment [X] for [Y]" (high-engagement plays, use sparingly)

Don't stack CTAs. One ask.

### Optional: bonus tweet

Many threads add a P.S.-style bonus tweet after the "real" ending. Works if:
- It's genuinely additive, not duplicate content
- It's a self-promo that feels earned (newsletter, related thread, product)

## Thread length guidance

- **Shorter threads (5-8 tweets):** for tactical, listy content. Easier to finish.
- **Medium (9-15 tweets):** for story arcs, multi-step breakdowns.
- **Long (15+ tweets):** risky — most people drop off. Only if the content genuinely needs it.

Be ruthless about cutting. If a tweet doesn't earn its place, delete it.

## Voice rules

- Write like you talk — contractions, short sentences, occasional sentence fragments for rhythm.
- "You" in every tweet when possible.
- Concrete nouns, active verbs. Specifics over abstractions.
- Don't open with "So" or "I want to talk about."
- Second person directives work: "Stop doing X. Do Y instead."
- Numbers land harder than vague quantifiers. "Cut my email time from 2 hours to 20 minutes" beats "saved a lot of time."

## What to avoid

- Thread announcements ("Get comfortable, this is a long one")
- Constant "(1/12)" numbering (let the UI handle it unless the user specifically wants it)
- Padded filler tweets to make the thread look longer
- Ending on "What do you think?" as lazy engagement bait
- Over-using emojis (one or two accents at most — emoji walls read as low-effort)
- Walls of hashtags (don't use hashtags in threads — feels spammy)
- Fake-intimate stories that are clearly fabricated

## Output format

Deliver the thread as numbered tweets, each clearly separated:

```
TWEET 1 (hook):
[Copy]

TWEET 2:
[Copy]

TWEET 3:
[Copy]

...

TWEET [final] (close + CTA):
[Copy]
```

Then deliver separately:

1. **Character count per tweet** — flag any over 280
2. **3 alternative hook tweets** (Tweet 1 variants) — hook is the highest-leverage experiment
3. **Posting notes** — best time, whether to quote-tweet the first tweet later, whether to pin
4. **Repurposing suggestions** — how this thread could become a LinkedIn post, a blog post, or a YouTube Short
